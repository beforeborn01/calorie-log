# 2. 引言

## 2.1 项目背景

随着健康意识的提升和健身需求的普及，越来越多用户希望通过科学管理每日饮食摄入，实现增肌塑型、减脂增肌等个性化目标。当前现有饮食记录工具多存在功能单一、目标适配性差、营养素追踪不全面等问题，无法满足用户精准管理饮食、贴合健身目标的核心需求。为解决这一痛点，开发一款专注于饮食热量与各类营养素记录、支持个性化健身目标设定的APP（命名“食养记”），帮助用户科学规划饮食，高效达成健身目标。

## 2.2 文档目的

本文档作为“食养记”APP开发的基础需求依据，明确APP的核心功能、非功能需求、用户场景及基础交互逻辑，为后续设计、开发、测试及迭代提供统一基准，同时便于需求方逐步补充完善细节，确保开发成果符合预期。

## 2.3 目标读者

本文档目标读者包括产品经理、开发工程师、测试工程师、UI/UX设计师及需求方，其中开发工程师重点关注功能实现逻辑，设计师关注用户交互与视觉呈现，需求方重点关注核心功能适配性。

## 2.4 范围

### 2.4.1 项目目标

开发一款操作便捷、功能实用的饮食记录APP（食养记），核心实现每日饮食热量、各类营养素（蛋白质、碳水化合物、脂肪等）的精准记录与统计，支持增肌塑型、减脂增肌两种目标设定，结合用户基础信息生成个性化饮食目标，助力用户科学管理饮食，达成健身目标。

### 2.4.2 主要功能

核心功能包括用户注册登录、个人基础信息设置、健身目标（增肌塑型/减脂增肌）设定、每日饮食记录（含热量及营养素）、饮食数据统计与分析、基础设置、社交互动等，后续可根据需求补充扩展功能。

### 2.4.3 范围界定

包含内容：用户管理、目标设定、饮食记录、热量及营养素统计、数据可视化展示、基础设置、社交互动；不包含内容：暂不涉及复杂的第三方工具深度集成（除小米运动健康数据导入外），后续可根据需求迭代添加。

## 2.5 定义、首字母缩写词和缩略语

1.  热量：指食物提供的能量，单位为千卡（kcal）或千焦（kj）；

2.  营养素：本文档重点涵盖蛋白质、碳水化合物、脂肪、膳食纤维、维生素（基础类）、矿物质（基础类）；

3.  增肌塑型：以增加肌肉量、塑造身体线条为目标，需保证充足热量及蛋白质摄入；

4.  减脂增肌：以减少脂肪、保留/增加肌肉为目标，需控制总热量摄入，保证蛋白质占比，合理分配碳水与脂肪。

5.  基础代谢率（BMR）：指人体在清醒而又极端安静的状态下，不受肌肉活动、环境温度、食物及精神紧张等影响时的能量代谢率，是计算每日总能量消耗（TDEE）的基础，单位为千卡（kcal）。

6.  热量缺口：每日饮食摄入总热量＜每日总能量消耗（TDEE），用于减脂增肌目标；热量盈余：每日饮食摄入总热量＞每日总能量消耗（TDEE），用于增肌塑型目标。

7.  推荐摄入量（RNI）：指可以满足某一特定性别、年龄及生理状况群体中绝大多数个体（97%~98%）需要量的某种营养素摄入水平，是制定个人营养素摄入目标的重要依据。

8.  适宜摄入量（AI）：当某种营养素的推荐摄入量（RNI）无法确定时，通过观察或实验获得的健康人群某种营养素的摄入量，作为该营养素的摄入参考标准。

9.  饮食评分：基于《中国居民膳食指南（2022）》及用户个性化健身目标，从热量达标、营养素摄入、餐次分配、食物多样性四个维度，对用户每日饮食情况进行量化评分，直观反映饮食健康及适配性。

10. 健康烹饪方法：参考《中国居民膳食指南（2022）》，指低油、低盐、低糖、少油炸、多蒸煮凉拌，能最大程度保留食材营养成分，同时适配用户健身目标的烹饪方式（如减脂增肌适配蒸、煮、凉拌，增肌塑型适配清炒、烤制）。

11. 毛重：指食物未去除不可食用部分（如玉米外壳、虾壳、虾线、果皮等）的总重量，单位为克（g）。

12. 净重：指食物去除不可食用部分后，可食用部分的重量，单位为克（g），是计算食物热量及营养素含量的核心依据。

13. 净毛重比例：指某类食物净重与毛重的比值，由权威食材数据库提供，根据食物品种、成熟度等略有差异，可手动调整。

14. 小米运动健康数据：指小米运动健康APP记录的用户相关健康数据，核心包括运动数据、睡眠数据，其中运动数据含运动类型、运动时长、运动强度、心率、运动消耗热量等；睡眠数据含睡眠时长、深睡时长、浅睡时长、入睡时间、醒来次数等。

15. 皮质醇：一种应激激素，其水平与睡眠质量、运动强度、作息规律相关，本文档中通过睡眠数据、运动数据及用户基础信息，结合权威计算公式估算皮质醇水平，用于评估用户身体状态，辅助调整饮食及运动计划。

16. 训练强度：用于量化用户运动负荷的指标，结合运动类型、运动时长、心率数据计算（采用MET代谢当量法结合心率区间评估），分为低、中、高三个等级，辅助判断运动对热量消耗及身体状态的影响。

17. 体重：指用户身体的总重量，单位为千克（kg），是衡量身体状态、健身效果的核心指标之一，支持每日手动记录、定期追踪。

18. 体脂率：指身体脂肪重量占总体重的百分比（%），反映身体脂肪含量，辅助评估健身效果（如减脂、增肌），支持每日手动记录、定期追踪。

19. 多周期数据报表：包括日报、周报、月报、年报，用于拉长时间维度展示用户饮食、运动、睡眠、体重、体脂率等数据，通过汇总统计、趋势图表，帮助用户直观查看长期数据变化，评估健身及饮食管理效果。

20. 经验值：用户通过完成饮食记录相关操作获得的积分，是升级的核心依据，可通过每日饮食记录、达成饮食目标等行为获取，不可逆向扣除（除非违规，后续可扩展）。

21. 升级制度：基于经验值设定的用户等级体系，等级从低到高递增，升级后可解锁对应权益，用于提升用户粘性，鼓励用户坚持饮食记录。

22. 好友排名：基于用户经验值、饮食相关数据（如每日饮食评分、连续记录天数），在用户好友列表内进行的实时排名，支持多维度排序，增强社交互动性。

# 3. 总体描述

## 3.1 产品愿景

“食养记”致力于成为用户科学饮食管理的核心工具，以简单便捷的操作、精准的数据分析，贴合用户健身目标，帮助用户轻松记录饮食、掌握营养摄入，实现饮食与健身的高效结合，传递健康生活理念。

## 3.2 用户特征

目标用户主要为18-45岁，有明确健身目标（增肌塑型、减脂增肌）的人群，包括健身爱好者、健身新手、需要控制体重及营养摄入的人群；用户核心需求为精准记录饮食热量与营养素、获取个性化饮食目标、查看饮食数据趋势，对操作便捷性、数据准确性要求较高，具备基础手机操作能力。

## 3.3 运行环境

### 3.3.1 硬件环境

客户端设备：1. 移动端：智能手机（iOS、Android系统），支持摄像头（用于食物扫码、拍照识别，核心用于食物拍照识别功能）、网络连接；2. 网页版：支持主流浏览器（Chrome、Edge、Firefox、Safari）的电脑设备，支持网络连接，摄像头（用于扫码、拍照识别，需用户授权）；

服务器：满足APP正常运行、数据存储与交互，支持多用户同时在线操作。

### 3.3.2 软件环境

iOS系统：支持iOS 13.0及以上版本；

网页版浏览器：支持Chrome 90.0及以上、Edge 90.0及以上、Firefox 88.0及以上、Safari 14.0及以上版本，需开启JavaScript、Cookie功能，支持摄像头授权（用于扫码、拍照识别）；

Android系统：支持Android 10.0及以上版本；

数据库：支持用户数据、食物数据、饮食记录数据的安全存储与快速查询。

## 3.4 设计和实现约束

1.  技术选型需符合移动端APP、网页版开发标准，保证两端操作流畅性，避免卡顿、页面错乱；

2.  食物热量及营养素数据需准确，参考权威食物营养数据库，确保数据可靠性；

3.  需遵循用户数据隐私保护相关要求，加密存储用户个人信息及饮食记录，网页版需保障登录安全、数据传输安全；

4.  界面设计简洁直观，操作流程便捷，降低用户学习成本，移动端符合移动端交互习惯，网页版适配电脑端操作逻辑（如鼠标操作、键盘快捷键支持），支持响应式设计，适配不同屏幕尺寸；

5.  移动端与网页版数据实时同步，确保用户在不同设备登录后，饮食记录、目标设置、历史数据、好友关系、经验值、训练日/休息日设置等信息一致，操作同步生效；支持微信第三方登录，确保移动端授权、网页版扫码授权流程顺畅，账号关联准确。

2.  食物热量及营养素数据需准确，参考权威食物营养数据库，确保数据可靠性；

3.  需遵循用户数据隐私保护相关要求，加密存储用户个人信息及饮食记录；

4.  界面设计简洁直观，操作流程便捷，降低用户学习成本，符合移动端交互习惯。

## 3.5 假设与依赖

1.  假设用户已具备基础手机操作能力，能够独立完成饮食记录、目标设置等基础操作；

2.  依赖权威食物营养数据库，确保食物热量及营养素数据的准确性与完整性；

3.  依赖稳定的网络环境，确保用户数据同步、食物数据查询等功能正常使用。

4.  依赖Mifflin-St Jeor方程的准确性，确保BMR及TDEE计算结果科学可靠，贴合用户实际情况。

5.  依赖《中国居民膳食指南（2022）》中的营养素推荐摄入量（RNI）、适宜摄入量（AI）及饮食原则，确保每日营养素摄入量建议、饮食打分及优化建议的科学性、权威性。

6.  依赖权威食材烹饪数据库，确保烹饪方法的实用性、健康性，以及烹饪后食材热量、营养素数据的准确性，贴合健身目标需求。

7.  依赖权威食材净毛重比例数据库，预设常见难称重食物（如玉米、虾等）的净毛重比例，确保毛重转净重计算结果的准确性，贴合实际食用场景。

8.  依赖小米运动健康开放API，确保可正常获取用户授权的运动数据、睡眠数据，保障数据导入的稳定性与准确性。

9.  依赖权威皮质醇估算模型、训练强度计算模型（MET代谢当量法），结合小米运动健康导入的数据及用户基础信息，确保皮质醇水平、训练强度计算结果科学可靠。

10. 依赖用户好友关系数据库，确保好友添加、好友列表管理及排名数据的准确性、实时性。

11. 依赖用户好友关系数据库，确保好友添加、好友列表管理及排名数据的准确性、实时性。

12. 依赖网页版浏览器兼容性支持，确保扫码、拍照识别、图片上传、数据展示等功能在主流浏览器中正常运行，无兼容性问题。

13. 依赖移动端与网页版数据同步机制，确保两端数据实时互通、一致，避免数据丢失或不同步。

14. 依赖用户设置的训练日、休息日及训练强度信息，确保训练日/休息日的TDEE、目标热量及营养素比例计算准确，贴合当日身体消耗需求。

15. 依赖训练强度与活动系数增量的对应关系设定，确保训练日TDEE计算科学合理，匹配不同训练强度的能量消耗差异。

16. 依赖预设的力量训练动作库，确保动作类型分类清晰、覆盖常见力量训练场景，同时支持用户自定义动作，满足个性化训练记录需求。

17. 依赖力量训练数据与训练日、训练强度的关联机制，确保力量训练记录仅在训练日可操作，且训练数据能准确关联TDEE计算及饮食优化建议调整。

# 4. 详细需求

## 4.1 功能需求

### 4.1.1 用户管理模块

功能ID：FR-USER-001

功能名称：用户注册

所属模块：用户管理

功能描述：允许新用户通过手机号、邮箱、微信第三方账号完成注册，获取“食养记”APP及网页版使用权限，微信登录支持移动端APP微信授权、网页版微信扫码授权。

前置条件：用户未注册过本APP账号，设备联网。

输入：手机号/邮箱、验证码、密码（符合密码规范）；或微信授权指令（移动端）、微信扫码授权（网页版）。

输出：注册成功提示，跳转至登录页面或个人信息完善页面。

功能ID：FR-USER-002

功能名称：用户登录

所属模块：用户管理

功能描述：允许已注册用户通过账号密码、验证码、微信第三方账号登录APP及网页版，获取个人账号权限；微信登录支持移动端APP微信授权、网页版微信扫码授权，首次微信登录需绑定手机号完成账号关联。

前置条件：用户已完成注册，设备联网。

输入：手机号/邮箱、密码（或验证码）；或微信授权指令（移动端）、微信扫码授权（网页版）。

输出：登录成功提示，跳转至APP首页；登录失败提示错误原因（如密码错误、账号不存在）。

功能ID：FR-USER-003

功能名称：个人基础信息设置

所属模块：用户管理

功能描述：用户登录后，可完善个人基础信息，为目标热量计算提供依据。

前置条件：用户已登录APP。

输入：性别、年龄、身高（cm）、体重（kg）、日常活动量（极少运动/轻度运动/中度运动/高强度运动）。

输出：信息保存成功提示，信息同步至系统，用于后续目标计算。

### 4.1.2 目标设定模块

功能ID：FR-TARGET-001

功能名称：健身目标选择

所属模块：目标设定

功能描述：用户可选择两种核心健身目标——增肌塑型、减脂增肌，支持目标切换（切换后自动重新计算目标热量及营养素比例）。

前置条件：用户已完善个人基础信息。

输入：目标类型选择（增肌塑型/减脂增肌）。

输出：目标设置成功提示，系统根据个人信息、目标类型及训练日/休息日设置，自动计算每日（训练日/休息日）目标热量及营养素比例。

功能ID：FR-TARGET-004

功能名称：训练日与休息日设置

所属模块：目标设定

功能描述：用户可设置每周训练日与休息日，支持两种设置方式（固定日期设置：选择每周固定几天为训练日，其余为休息日；自定义日期设置：手动选择每日为训练日/休息日），支持设置训练日的训练强度（低强度/中强度/高强度），用于精准调整训练日TDEE计算的活动系数增量；支持随时修改训练日/休息日设置及训练强度，修改后系统自动重新计算对应日期的TDEE、目标热量及营养素比例，同步更新所有相关页面数据。训练日与休息日设置同步适配移动端与网页版，两端数据实时同步。

前置条件：用户已完善个人基础信息，已选择健身目标。

输入：训练日/休息日设置方式、具体日期选择、训练日训练强度（低/中/高）、修改指令（可选）。

输出：设置/修改成功提示，训练日与休息日列表同步显示在目标设置页面，系统自动关联后续TDEE及目标热量计算；修改后，历史已计算的对应日期数据可选择是否同步更新（默认不更新，用户可手动触发更新）。

功能ID：FR-TARGET-002

功能名称：目标热量及营养素设置

所属模块：目标设定

功能描述：系统根据用户个人信息（性别、年龄、身高、体重、活动量）、所选目标，以及用户设定的训练日/休息日，分别计算每日基础代谢率（BMR）、每日总能量消耗（TDEE）及目标热量、营养素比例。其中BMR采用Mifflin-St Jeor方程计算（男性：BMR = 10×体重(kg) + 6.25×身高(cm) - 5×年龄 - 161；女性：BMR = 10×体重(kg) + 6.25×身高(cm) - 5×年龄 + 5）；TDEE根据训练日/休息日采用不同活动系数计算：基础活动系数（对应日常活动量）为极少运动：1.2、轻度运动：1.375、中度运动：1.55、高强度运动：1.725；训练日在基础活动系数基础上增加0.1~0.2（根据训练强度适配，低强度训练+0.1，中高强度训练+0.2），休息日在基础活动系数基础上减少0.1，确保TDEE贴合当日身体消耗；目标热量根据所选目标及当日日期类型（训练日/休息日）调整（增肌塑型：训练日TDEE + 15%-20%、休息日TDEE + 10%-15%，均为热量盈余；减脂增肌：训练日TDEE - 10%-15%、休息日TDEE - 15%-20%，均为热量缺口）。营养素比例结合日期类型微调（训练日：蛋白质占比提升5%，碳水占比提升5%，脂肪占比下降10%；休息日：蛋白质占比维持基础值，碳水占比下降5%，脂肪占比提升5%），基础营养素比例（增肌塑型：蛋白质20%-30%、碳水40%-50%、脂肪20%-30%；减脂增肌：蛋白质25%-35%、碳水30%-40%、脂肪20%-25%），用户可在系统计算值基础上，手动微调目标范围（需提示合理调整区间）。

前置条件：用户已选择健身目标，且已设置训练日与休息日。

输入：手动微调的目标热量、各营养素占比（可选）；训练日/休息日的训练强度（可选，用于调整活动系数增量）。

输出：目标保存成功，首页及统计页面同步显示当日（训练日/休息日）目标数据；系统自动区分日期类型，每日实时更新对应TDEE及目标热量、营养素比例。

功能ID：FR-TARGET-003

功能名称：每餐热量及每日营养素摄入量建议

所属模块：目标设定

功能描述：系统结合《中国居民膳食指南（2022）》标准、用户个人基础信息（性别、年龄、体重等）及健身目标，生成每日营养素摄入量建议和每餐热量分配建议。每日营养素摄入量参考指南推荐值（RNI/AI），结合健身目标调整：蛋白质65~85g/d（增肌塑型可提升至80~100g/d）、碳水化合物250~300g/d（添加糖摄入量不超过50g/d）、脂肪25~30g/d（烹调用油占比不超过30g/d）、膳食纤维25~30g/d，维生素及矿物质参考指南对应人群推荐值；每餐热量分配参考指南餐次原则，结合用户每日目标热量拆分：早餐占比20%~30%、午餐占比40%~50%、晚餐占比20%~30%、加餐（可选）占比5%~10%，明确每餐建议热量范围，同步显示每餐营养素摄入量参考值。

前置条件：用户已完善个人基础信息、设置健身目标，且已设置训练日与休息日。

输入：无（自动调用用户个人信息、健身目标及《中国居民膳食指南（2022）》标准数据）。

输出：每日各营养素摄入量建议（含RNI/AI参考值及目标适配值，区分训练日/休息日）、每餐热量分配建议（含具体热量范围，训练日午餐/加餐热量占比可适当提升，休息日适当降低），同步显示在目标设置页面及首页，支持随时查看，系统自动根据当日日期类型切换显示对应建议。
